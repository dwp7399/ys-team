# 方法论架构师 经验记忆

## 角色记忆文件路径（2026-04-15）

**错误模式**：skill 文案中硬编码英文 role-id 路径（如 `roles/spec-reviewer.md`），但本仓角色名用中文持久化，导致 full-auto 链路出现路径漂移、记忆读写失败。

**正确做法**：凡涉及角色记忆路径的 skill 文案，必须标注"本仓优先用中文文件名，英文 role-id 作为回退"；执行前先 `ls .ys_team/memory/` 确认实际文件名。

**适用场景**：所有读写 `.ys_team/memory/` 的环节（spec-talk、spec-work、qa、close）；本仓角色名为中文时必须遵守。

## Wrapper-only 重构模式（2026-05-07）

**场景**：在 SKILL.md / 文档型文件加结构性标签（如 `<what-to-do>` / `<supporting-info>`）时，若 AC 约束为"diff 仅新增、不删除原有内容、不重排原有章节"，常规重写或抽取式分块都会产生删除行。

**正确做法**：把所有原有内容连续段整块包到 `<supporting-info>` 内（从原首段开始到原末行结束），`<what-to-do>` 作为独立新增块插入到 `# title` 之后、`<supporting-info>` 之前。`<what-to-do>` 用「按下方 ××」指针引用 `<supporting-info>` 内的既有章节，不复制不抽取。这样 diff 中除增强项（如 description 替换）外只有 `+` 行，AC-04 类约束自动满足。

**适用场景**：spec 要求"原有信息逐字保留"或"diff 仅新增"的文档结构重构；外部模式吸收（mattpocock-style 分块、其他 skill 写作约定）落到本仓既有 SKILL.md 时。
