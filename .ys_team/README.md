# ys-team 本地配置

本目录是 `ys-team` 仓库自己的本地工作流基线。

目的：

- 用 ys-team 自己管理 ys-team
- 保持方法论、skills、README 和 spec 在同一套原则下演进
- 让本仓后续改造有明确 spec、验收和文档同步约束
- 对齐 `examples/baseline/.ys_team/` 的稳定版本约束，同时保留方法仓自己的角色和边界表达

使用顺序：

1. 先读 `team.md`
2. 再读 `methods.md`
3. 再读 `policy.md`
4. 再读 `delivery-flow.md`
5. 起草或更新 spec 时，使用 `templates/`
6. 当 skills、baseline 或仓库结构明显变化时，同步更新 `docs/project/module-index.md`
