# Methods

## Principle

默认工作流采用“弱引导、强结果”。

含义：

- 先理解当前是在讨论需求、起草 spec、执行工作，还是做验收判断。
- 不把所有对话强行套进固定流程图。
- 每轮讨论都要收敛到明确结论、现状和下一步。

## Result Card

每轮重要讨论至少应收敛到：

- `Current Mode`
- `Decision`
- `Current State`
- `Why`
- `Needed Changes`
- `Next Step`

## Role Brief

每轮讨论默认给出角色简报，说明当前判断、风险和下一步。

## Decision Meanings

- `PASS`
- `BLOCKED`
- `REJECT`

## External Escalation

如果 internal team 认为需要 external role：

1. 先停止继续扩展讨论
2. 明确提出 external support request
3. 等用户同意后再继续

## Execution Intent

当问题已经收敛到可执行程度时，默认应继续进入 spec / work / acceptance 链，而不是停在抽象讨论。

## Discussion Lens

项目可按场景装载思维透镜，例如：

- 第一性原理
- 苏格拉底提问
- 奥卡姆剃刀

它们用于帮助收敛，不构成固定流程。
